## Comentários (breve)
**01.** HTTP é protocolo de camada de aplicação para web.
**02.** DNS resolve nomes (ex.: site.gov.br) em IP.
**03.** DHCP distribui IP, máscara, gateway, DNS etc.
**04.** IPv4 usa quatro octetos 0–255; 192.168.0.10 é válido.
**05.** Intranet é rede corporativa/privada interna.
**06.** SMTP envia e-mails entre servidores/clientes de envio.
**07.** IMAP/POP3 recebem mensagens no cliente de e-mail.
**08.** /24 (255.255.255.0) permite 254 hosts úteis (classe C típica).
**09.** Cat6 suporta Ethernet de maior desempenho que Cat5e.
**10.** HTTPS = HTTP + criptografia TLS/SSL.
**11.** Firewall filtra tráfego conforme políticas.
**12.** FTP usa porta 21 (canal de controle).
**13.** Wi-Fi = redes sem fio IEEE 802.11.
**14.** Roteador encaminha pacotes entre redes (camada 3).
**15.** Ping usa ICMP Echo Request/Reply.
**16.** VPN cria túnel seguro para acesso remoto.
**17.** MAC identifica interface física/lógica de rede (camada 2).
**18.** Proxy intermediária requisições e aplica filtros.
**19.** Latência = delay/tempo de ida e volta.
**20.** TCP oferece confiabilidade, controle de fluxo e ordem.
