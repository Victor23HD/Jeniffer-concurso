# Informática 02 — Windows e Linux

## Conceitos comuns

| Termo | Significado |
|---|---|
| Arquivo | conjunto de dados com nome e extensão (.docx, .xlsx, .pdf) |
| Pasta / diretório | container de arquivos e outras pastas |
| Atalho | ponteiro para um arquivo/programa (não é o arquivo em si) |
| Área de trabalho | tela inicial com ícones |
| Área de transferência | o que foi copiado/recortado (Ctrl+C / Ctrl+X) |

Extensão indica o tipo; ocultar extensões pode ser risco de segurança (`foto.jpg.exe`).

## Windows (versão recente — ideias cobradas)

- Explorador de Arquivos: pastas, busca, copiar/mover  
- Painel de Controle / Configurações: rede, contas, dispositivos  
- Gerenciador de Tarefas: Ctrl+Shift+Esc — processos, desempenho  
- Prompt / PowerShell: linha de comando  
- Lixeira: exclusão lógica; esvaziar remove de forma mais definitiva  
- Atalhos úteis:  
  - Ctrl+C / Ctrl+V / Ctrl+X / Ctrl+Z  
  - Alt+Tab (alternar janelas)  
  - Win+L (bloquear) · Win+D (área de trabalho) · Win+E (explorador)  
  - Ctrl+Shift+Esc (tarefas)

Permissões e contas: administrador × usuário padrão.

## Linux (visão de concurso)

- Sistema de arquivos hierárquico a partir de `/`  
- Pasta do usuário costuma ser `/home/nome`  
- Terminal (bash): comandos comuns  
  - `ls` listar · `cd` entrar · `pwd` onde estou  
  - `cp` copiar · `mv` mover/renomear · `rm` apagar  
  - `mkdir` criar pasta · `sudo` admin  
- Permissões: leitura (r), escrita (w), execução (x) para dono/grupo/outros  
- Distribuições: Ubuntu, Debian, Fedora… (todas são Linux)

## Gerenciamento de arquivos (boa prática)

- Nomes claros e pastas organizadas  
- Backup antes de apagar em massa  
- Não executar arquivos de origem duvidosa

## Checklist

- [ ] Atalho ≠ arquivo original
- [ ] Sei atalhos básicos do Windows
- [ ] No Linux, `/` é a raiz; `sudo` eleva privilégio
